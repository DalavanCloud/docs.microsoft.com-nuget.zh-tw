<span data-ttu-id="0d8bb-418">本章節描述一種用戶端概念，雖然不一定是託管的通訊協定，但應該是任何實用的類別目錄用戶端實作的一部分。</span><span class="sxs-lookup"><span data-stu-id="0d8bb-418">This section describes a client concept that, although is not necessarily mandated by the protocol, should be part of any practical catalog client implementation.</span></span>

<span data-ttu-id="0d8bb-419">因為此目錄是以時間編製索引的僅附加的資料結構，應儲存在用戶端**游標**代表最多哪個時間點的用戶端已在本機處理類別目錄項目。</span><span class="sxs-lookup"><span data-stu-id="0d8bb-419">Because the catalog is an append-only data structure indexed by time, the client should store a **cursor** locally, representing up to what point in time the client has processed catalog items.</span></span> <span data-ttu-id="0d8bb-420">請注意，永遠不會使用用戶端的電腦時鐘會產生此資料指標的值。</span><span class="sxs-lookup"><span data-stu-id="0d8bb-420">Note that this cursor value should never be generated using the client's machine clock.</span></span> <span data-ttu-id="0d8bb-421">相反地，值應該來自於目錄物件的`commitTimestamp`值。</span><span class="sxs-lookup"><span data-stu-id="0d8bb-421">Instead, the value should come from a catalog object's `commitTimestamp` value.</span></span>

<span data-ttu-id="0d8bb-422">每次用戶端想要處理新的套件來源上的事件，它需要查詢類別目錄的所有類別目錄項目，認可時間戳記大於其預存的資料指標。</span><span class="sxs-lookup"><span data-stu-id="0d8bb-422">Every time the client wants to process new events on the package source, it need only query the catalog for all catalog items with a commit timestamp greater than its stored cursor.</span></span> <span data-ttu-id="0d8bb-423">用戶端已順利處理所有新的類別目錄項目之後，它會記錄只會處理做為其新的資料指標值的類別目錄項目的最新的認可時間戳記。</span><span class="sxs-lookup"><span data-stu-id="0d8bb-423">After the client successfully processes all new catalog items, it records the latest commit timestamp of catalog items just processed as its new cursor value.</span></span>

<span data-ttu-id="0d8bb-424">使用這個方法，用戶端可以確定永遠不會遺漏任何發生的封裝來源的封裝事件。</span><span class="sxs-lookup"><span data-stu-id="0d8bb-424">Using this approach, the client can be sure to never miss any package events that occurred on the package source.</span></span>
<span data-ttu-id="0d8bb-425">此外，用戶端永遠不會有重新處理資料指標的記錄的認可時間戳記之前的舊事件。</span><span class="sxs-lookup"><span data-stu-id="0d8bb-425">Additionally, the client never has to reprocess old events prior to the cursor's recorded commit timestamp.</span></span>

<span data-ttu-id="0d8bb-426">資料指標的強大概念用於許多 nuget.org 背景工作，用來保持 V3 API 本身。</span><span class="sxs-lookup"><span data-stu-id="0d8bb-426">This powerful concept of cursors is used for many of nuget.org background jobs and is used to keep the V3 API itself up-to-date.</span></span>
